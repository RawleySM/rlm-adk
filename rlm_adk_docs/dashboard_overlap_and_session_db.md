## Part 2: Two session.db Files Investigation

### Root Cause: Relative Path Resolution Based on CWD

| File | Size | Modified | Created By |
|------|------|----------|------------|
| `.adk/session.db` | 4.1 KB | Feb 20 | `adk run rlm_adk` from project root |
| `rlm_adk/.adk/session.db` | 4.3 MB | Feb 24 | Module/dashboard launch from package dir |

### What Code Creates session.db?

**RLM ADK Wrapper** (`rlm_adk/agent.py` lines 77-108):
```python
_DEFAULT_DB_PATH = ".adk/session.db"  # Line 64 â€” relative to CWD!

def _default_session_service(db_path: str | None = None):
    resolved_path = db_path or os.getenv("RLM_SESSION_DB", _DEFAULT_DB_PATH)
    db_dir = Path(resolved_path).parent
    db_dir.mkdir(parents=True, exist_ok=True)
    conn = sqlite3.connect(resolved_path)  # Resolves relative to CWD
    # ... applies WAL mode + performance pragmas
    return SqliteSessionService(db_path=resolved_path)
```

**ADK Framework** (`.venv/.../google/adk/sessions/sqlite_session_service.py`):
- Creates database on first `create_session()` call
- Applies schema and pragmas

### Path Resolution: CWD-Dependent

**Scenario A: `adk run rlm_adk` from project root**
```bash
$ cd /home/rawley-stanhope/dev/rlm-adk
$ adk run rlm_adk
# CWD = /home/rawley-stanhope/dev/rlm-adk
# Creates: /home/rawley-stanhope/dev/rlm-adk/.adk/session.db (4.1 KB)
```

**Scenario B: Module/dashboard from package dir**
```bash
$ cd /home/rawley-stanhope/dev/rlm-adk/rlm_adk
$ python -m rlm_adk  # or dashboard launch
# CWD = /home/rawley-stanhope/dev/rlm-adk/rlm_adk
# Creates: /home/rawley-stanhope/dev/rlm-adk/rlm_adk/.adk/session.db (4.3 MB)
```

### ADK CLI vs RLM ADK Session Service

**ADK CLI path** (`.venv/.../google/adk/cli/utils/local_storage.py`):
- Uses `PerAgentDatabaseSessionService` with per-agent storage
- Creates: `<agents_root>/<app_name>/.adk/session.db`

**RLM ADK bypasses this:**
- `create_rlm_runner()` calls `_default_session_service()` (line 442)
- Uses RLM's own path resolution, not ADK's per-agent factory
- Result: path depends entirely on CWD

### Is One from Dead Code?

**No.** Both are actively created by different invocation contexts. The root `.adk/session.db` (4.1 KB, older) is from CLI invocations from project root. The nested `rlm_adk/.adk/session.db` (4.3 MB, recent) is from module/dashboard invocations with CWD inside the package.

### Same Issue Affects Other Files

All `.adk/` outputs use relative paths and are subject to the same CWD-split:

| File | Purpose | Same CWD Issue? |
|------|---------|-----------------|
| `.adk/session.db` | Session state | YES |
| `.adk/traces.db` | OTel traces | YES |
| `.adk/context_snapshots.jsonl` | Context windows | YES |
| `.adk/model_outputs.jsonl` | Model outputs | YES |
| `.adk/artifacts/` | Artifacts | YES |

### Recommendations

1. **Set env var for consistency**: `export RLM_SESSION_DB=/home/rawley-stanhope/dev/rlm-adk/.adk/session.db`
2. **Always run from project root** (recommended workflow)
3. **Consider absolute path resolution**: Resolve `_DEFAULT_DB_PATH` relative to `__file__` instead of CWD
4. **For tests**: Use `InMemorySessionService` (already done â€” no session.db created)
