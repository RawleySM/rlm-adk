        ## Validation Report: EH-004

### Summary
**PASS** — The `on_replay_change` handler is correctly defined, wired to the dropdown's `on_change`, calls `controller.set_replay_path(e.value)`, and the controller method enforces mutual exclusion with `selected_provider_fake_fixture` and clears `launch_error`.

### Evidence

1. **Handler definition** — `live_app.py:396-398`: `def on_replay_change(e)` calls `controller.set_replay_path(str(e.value or ""))` then `live_ui.refresh_all()`. The `str(e.value or "")` coercion correctly handles `None` (dropdown cleared) by converting to `""`.

2. **Dropdown wiring** — `live_app.py:472-478`: `ui.select(...)` with `on_change=on_replay_change` binds the handler to the replay fixture dropdown. The dropdown's initial value is set from `controller.state.replay_path or None`.

3. **Controller mutual exclusion** — `live_controller.py:75-79`: `set_replay_path()` sets `self.state.replay_path = replay_path`, conditionally clears `selected_provider_fake_fixture` to `""` when `replay_path` is truthy, and unconditionally sets `self.state.launch_error = None`.

4. **State model fields** — `live_models.py:326`: `replay_path: str = ""`, `live_models.py:330`: `launch_error: str | None = None`, `live_models.py:338`: `selected_provider_fake_fixture: str = ""`. All fields have correct default values matching the expected cleared state.

5. **Symmetry with provider-fake handler** — `live_controller.py:81-85`: `set_provider_fake_fixture()` mirrors the mutual exclusion pattern (clears `replay_path` when a fixture is set), confirming the bidirectional exclusion design.

### Bugs Found

- **Bug:** Mutual exclusion in `set_replay_path` is conditional on truthiness (`if replay_path:`), meaning calling `set_replay_path("some_path")` clears `selected_provider_fake_fixture`, but calling `set_replay_path("")` does **not** clear it. This is arguably correct behavior (clearing the dropdown shouldn't wipe the other dropdown's selection), but differs from what a naïve reader might expect.
- **Location:** `live_controller.py:77-78`
- **Severity:** cosmetic (behavior is intentional — clearing one dropdown should not forcibly clear the other; only *selecting* a value triggers exclusion)
- **Fix:** No fix needed — current behavior is the correct UX pattern. Document the conditional exclusion if needed.

### Validation Notes

Static validation confirms the full event handler chain: NiceGUI `ui.select` dropdown → `on_replay_change` closure → `controller.set_replay_path()` → state mutation with mutual exclusion. The handler correctly coerces `None` to `""` via `str(e.value or "")`, preventing `None` from leaking into the state model's `str`-typed `replay_path` field. The `live_ui.refresh_all()` call after state mutation ensures the UI reflects the updated state. The only caveat is that this is a static analysis — runtime behavior of the NiceGUI `on_change` event delivery cannot be verified without running the dashboard.
