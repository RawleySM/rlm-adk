### Validation Notes

The `on_expand_child` event handler is **structurally sound**: the parameter is declared with correct defaults (`None`) in all three layers (`render_flow_child_card`, `render_flow_transcript`, `_render_block`), the truthiness guard at `flow_connectors.py:134` correctly prevents the button from rendering when the callback is absent, and the lambda at line 137 correctly passes the `FlowChildCard` object to the callback.
